# Contributing to Soundwave

Thank you for contributing to Soundwave.

## How to Contribute

1. Fork the repository and create a feature branch.
2. Keep changes focused and scoped to a single issue.
3. Update documentation when behavior or setup changes.
4. Open a pull request with a clear summary of changes.

## Pull Request Guidelines

- Use descriptive commit messages.
- Keep pull requests small and reviewable.
- Include validation notes for any code changes.
- Do not include secrets, credentials, or local artifacts.

## Reporting Issues

When opening an issue, include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Environment details (OS/device/audio stack)
